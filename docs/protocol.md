# Multi-AI Team 메시지 교환 프로토콜 (v2.0)

이 문서는 Multi-AI Team 시스템 내에서 AI 에이전트 간의 메시지 교환 및 협업을 위한 프로토콜 규격을 정의합니다.

## 1. 메시지 유형

### 1.1 사용자 명령 (User Command)
사용자가 `chat_loop`를 통해 전달하는 입력입니다.
- **형식**: 일반 텍스트 또는 `@mention` 포함 텍스트
- **전달 방식**: Tmux `send-keys`를 통해 각 AI의 대화형 CLI 패인으로 직접 전달

### 1.2 시스템 브로드캐스트 (System Broadcast)
시스템이 모든 AI에게 공지하거나 지시하는 메시지입니다.
- **예**: "시스템을 시작합니다.", "모든 응답을 취합 중입니다."

### 1.3 AI 간 협업 메시지 (Orchestration Message)
`TaskOrchestrator` 또는 `BatchDiscussion`을 통해 전달되는 메시지입니다.

#### Phase 1: Planning (계획 수립)
- 각 AI는 주어진 작업에 대해 자신만의 계획 초안을 작성합니다.
- **출력 요구사항**: 접근 방식, 핵심 결정 사항, 잠재 리스크

#### Phase 2: Assignment (역할 할당)
- Claude(또는 주력 모델)가 모든 계획을 검토하고 각 AI에게 구체적인 작업을 할당합니다.
- **형식**: `[모델명] 지시사항`
- **예**: `[codex] 각 논점의 근거 자료를 정리하라.`

#### Phase 3: Execution (실행)
- 각 AI는 할당받은 지시사항을 자신의 패인에서 실행합니다.
- 실행 완료 시 터미널에 결과를 출력하며, 시스템은 화면 변화가 멈출 때까지 대기합니다.

## 2. 상태 표시 규격 (Status Indicators)

Tmux 패인 상단에 표시되는 AI의 현재 상태 정의입니다.

- **⏳ 대기중 (Waiting)**: AI가 입력을 기다리고 있는 유휴 상태
- **🚀 실행중 (Running)**: AI가 입력을 처리 중이거나 응답을 생성 중인 상태
- **✅ 완료 (Completed)**: 응답 생성이 완료되고 결과가 화면에 고정된 상태
- **❌ 에러 (Error)**: 실행 중 오류가 발생했거나 타임아웃이 발생한 상태

## 3. 응답 취합 및 요약 (Synthesis)

여러 AI의 응답을 하나로 통합하는 과정입니다.

1. **캡처**: 각 AI 패인의 마지막 N줄(기본 80줄)을 텍스트로 캡처
2. **전달**: 취합된 텍스트를 Claude에게 전달
3. **요약**: Claude는 다음 기준에 따라 요약본을 생성
    - 각 AI의 핵심 포인트 포함
    - 의견 충돌 지점 명시
    - 최종 통합 권장안 제시

## 4. @Mention 규칙

- `@model_name`: 특정 모델에게만 메시지를 전달합니다.
- 여러 명 언급 가능: `@claude @codex ...`
- `@all`: 명시적으로 전체 AI에게 전달합니다.
- 언급이 없을 경우: 활성화된 모든 모델에게 메시지가 **브로드캐스트**되며, 자동 요약(Auto-synthesis)이 활성화됩니다.
